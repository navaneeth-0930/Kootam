# 📖 Overview

**KOOTTAM** is a desktop-based academic collaboration platform built entirely in **Java**.

Inspired by platforms such as **Google Classroom**, **Microsoft Teams**, and **Discord**, KOOTTAM aims to simplify communication, assignment management, classroom organization, and academic collaboration between students and teachers.

Unlike a traditional college project, KOOTTAM is intentionally designed as a **Learning-by-Building** project. Every module is mapped to specific Java concepts, enabling the development team to learn Java progressively—from beginner fundamentals to industry-level software engineering practices.

# 👨‍💻 Team Structure

The project is divided into four independent modules to encourage parallel development and collaborative learning.

| Member | Module | Primary Learning |
|---------|----------|------------------|
| Member 1 | Authentication & User Management | OOP, Validation, Exception Handling |
| Member 2 | Rooms & Announcements | Collections, CRUD, Relationships |
| Member 3 | Assignments, Attendance & Reminders | Date API, Streams, File Handling |
| Member 4 | JavaFX, Database & Integration | JavaFX, JDBC, SQLite, MVC |

Each module is designed to be independently testable before integration.

---

# 🌱 Learning Philosophy

KOOTTAM follows a simple principle:

> **Learn by Building.**

Instead of learning Java concepts in isolation, every feature of the application introduces one or more programming concepts through practical implementation.

Examples:

- Login System → Classes, Objects, Encapsulation
- Room Management → Collections
- Assignments → Composition
- Attendance → File Handling
- Database → JDBC
- Dashboard → JavaFX
- Testing → JUnit
